Activate server-side push notifications
Step 1 — Deploy the edge function
In your terminal (or Replit shell):

npx supabase functions deploy push-notification-trigger --project-ref rhnsjqqtdzlkvqazfcbg

Step 2 — Generate a webhook token
Pick any strong random string (e.g. from openssl rand -hex 32 in terminal) or just paste a long UUID. You'll use it in both Step 3 and Step 4.

Step 3 — Add it to edge function secrets
Go to Supabase Dashboard → Edge Functions → push-notification-trigger → Secrets and add:

PUSH_WEBHOOK_TOKEN = <your-token-from-step-2>

Step 4 — Run both SQL migrations in the SQL Editor
Supabase Dashboard → SQL Editor — run these two files in order:

First: supabase/migrations/20260527_recreate_notifications.sql
(recreates the notifications + notification_preferences tables)

Then: supabase/migrations/20260527_message_push_trigger.sql
After pasting it, also run this one extra line with your actual token:

ALTER DATABASE postgres SET "app.push_webhook_token" = 'your-token-from-step-2';

That's it. Once those run, every new message INSERT fires a PostgreSQL trigger that calls the edge function via pg_net — which looks up the recipient's Expo push token, checks their push preferences, and delivers the notification through Expo's push gateway — all server-side, completely independent of whether any app is open.

The trigger also covers incoming calls (immediate high-priority push) and the notifications table (likes, follows, mentions, etc.) once that table is recreated.
