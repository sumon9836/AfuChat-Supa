In addition to the full startup audit, I need you to produce a Top Suspicious Startup Features Report.

Rank the startup features and modules that are most likely to cause an Android app to:

- Launch successfully.
- Display the splash screen.
- Immediately close with an Android crash notification.

Requirements

1. List the Top 20 most suspicious startup components.

2. Rank each component by probability:
   
   - Very High
   - High
   - Medium
   - Low

3. For each suspicious component, provide:
   
   - Feature or module name.
   - File(s) involved.
   - Why it is suspicious.
   - Typical Android crash symptoms.
   - Whether it can crash before the first screen renders.
   - Whether it involves native code.
   - How to temporarily disable it safely.
   - How to verify whether it is the root cause.

4. Pay special attention to:
   
   - app/_layout.tsx
   - Expo Router initialization.
   - Splash screen logic.
   - Root providers.
   - Authentication providers.
   - Font loading.
   - Theme providers.
   - Environment variables.
   - AsyncStorage and SecureStore.
   - Firebase or Supabase.
   - react-native-track-player.
   - react-native-worklets.
   - Reanimated.
   - Gesture Handler.
   - Push notifications.
   - Background tasks.
   - Deep linking.
   - Network requests during startup.
   - Custom hooks that execute on mount.
   - Native modules.
   - Android-specific initialization.

5. Create a table with:
   
   - Rank.
   - Component.
   - Risk Level.
   - Probability of causing the crash.
   - Recommended action.

6. Identify the Top 5 most likely causes and explain why they are stronger suspects than the others.

7. Recommend the exact order in which suspicious startup features should be disabled and tested, prioritizing those most likely to cause an immediate splash-screen crash.

8. Highlight any startup component that could fail silently in a production build, causing the app to close without showing a JavaScript error.

The goal is to prioritize the investigation by identifying the startup components most likely to be responsible for the Android splash-screen crash and provide a systematic testing order to isolate the root cause.
