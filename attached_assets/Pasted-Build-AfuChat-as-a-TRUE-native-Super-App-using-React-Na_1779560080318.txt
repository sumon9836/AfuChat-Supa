Build AfuChat as a TRUE native Super App using React Native + Expo architecture similar to WeChat and Binance.

IMPORTANT:
- Use React Native + Expo only
- Do NOT use web-based mini apps
- Do NOT use PWAs
- Do NOT use browser-style modules
- Everything must remain native inside AfuChat
- Every mini app must feel like a real standalone app
- UI/UX must be fully native and ultra smooth

MAIN GOAL:
Create a modular native Super App system where AfuChat can host multiple independent mini apps inside one application while keeping native performance and native UX.

TECH STACK:
- React Native
- Expo SDK
- Expo Router
- React Navigation
- Expo Dev Client
- Expo Updates
- Dynamic Imports
- Zustand or Redux
- Reanimated
- Gesture Handler
- Native Stack Navigation
- Expo Modules API

IMPORTANT:
Use Expo prebuild/eject only where native Android features are required such as:
- separate Android tasks
- taskAffinity
- custom activities
- advanced lifecycle handling

CORE ARCHITECTURE:
Create:
1. AfuChat Core App
2. Native Mini App Runtime
3. Dynamic Module Loader
4. Module Registry
5. Independent Navigation Runtime
6. Shared SDK Layer
7. Native Android Task Manager

MINI APP SYSTEM:
Mini apps must:
- behave like real native apps
- have independent navigation stacks
- preserve state independently
- restore state after reopen
- load dynamically
- unload from memory when inactive
- support deep linking
- support background persistence

DO NOT:
- use tabs as fake mini apps
- use web wrappers
- use iframe-like systems
- use heavy WebViews

EXAMPLE MINI APPS:
- AfuAI
- AfuPay
- AfuMarket
- AfuMusic
- AfuChannel
- AfuGames
- AfuBusiness
